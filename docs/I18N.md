# Localization Plan

dayby ships with English UI first, but product copy should be ready for Japanese and Korean.

## Initial Policy

- Keep all new user-facing strings short and context-aware.
- Avoid hard-coding cultural assumptions into dates, names, and group copy.
- Prefer product-neutral keys later, such as `post.success.title`, instead of screen-specific prose embedded in components.
- Localize date formatting with `Intl.DateTimeFormat` by locale.
- Keep the brand as `dayby` in every locale.

## Target Locales

- English: initial UI
- Japanese: early priority
- Korean: early priority

## Before Release

- Extract UI strings into a typed message catalog.
- Add locale detection from device settings.
- Add an in-app language override only if users ask for it during testing.
- Review Japanese and Korean copy manually; do not rely only on machine translation.
